# Directory structure

There are three basic directory: data for operations about data, arch for models, and trainer for training logic.

# arch

Store models, one file one model is recommended.

# data

Operations will include

  • provided data source (dataxy.py)
  • data augment (transform.py)
  • delegate (delegate.py), see DatasetBuilder#Delegate for details.
  • other operations

# trainer

# Change working directory

See initialized file _trainer.py for details. You will see that the working directory is changed into the root path of the project, which means that you can directly run this file.

# Mixin
