# More details

This post introduce more details about thexp.

# Params rules

When you use command line to give args to params and the key haven't contained in the Params instance, a warning will be raised, like

.....: UserWarning: xxx is a new param,please check your spelling. it's more recommended to define in advance.))
1

It is considered that spell mistakes might happen when type a command line , and if it happens, the scipt may still run normally, and with hyperparams you don't want. It may waste much time or computing resources, which is frustrating. That's why a warning will be raised when this thing happens.

# Environment variables

You can add some environment variables to control the action about Git in thexp.

THEXP_IGNORE_REPO # don't init Git repository
THEXP_COMMIT_DISABLE # don't commit this test
1
2

Params contains a variable git_commit, if the value is set to False, the Trainer will add THEXP_COMMIT_DISABLE to environment variable, so the commit operation will be disabled.

# Name rules

In Meter and Params, for achieving their features, there are many magic function were overrided, and the variable name need to follow a rule: no underline in each side of the variable name, or you will see some abnormal actions.

# gitignore extention

Files have below suffix will be added to gitignore automatically, in case spending much time on git operations.

*.pth
*.npy
*.ckpt
