# 细节

本节提供该框架的一些实现细节的介绍。

# Params 规范

当使用命令行参数接受超参时，如果出现了未提前在 params 中定义好的变量名，那么会抛出如下警告：

.....: UserWarning: xxx is a new param,please check your spelling. it's more recommended to define in advance.))
1

考虑到在试验过程中经常存在命令行参数敲错的情形，而这种错误有时并不会影响试验的正常运行（比如已经定义了默认参数，该命令行参数只是额外新增了一个变量），但会导致试验使用的超参数不符合预期，如果你的单次程序运行周期很长，出现这种问题会很令人沮丧（也有可能甚至无法发现这个错误）。因此，当通过命令行参数方式定义超参数时候，会对新设置的参数抛出警告用于提醒。

基于这个原因，强烈建议使用者在前定义好所有的超参数，这有利于命令行参数时的错误识别和一些其他因素。

# 环境变量控制

基于 thexp 运行的实验和 Git 控制息息相关，但该框架在开发时候受限于作者的强迫症，仍然贴心的考虑了没有 Git 或者暂时不想用 Git 的使用者。只需要在环境变量中添加相关的环境变量，就可以取消建立 Git 仓库或者禁止在实验运行开始时提交实验。

相关的环境变量为：

THEXP_IGNORE_REPO # 用于禁止建立 Git 仓库
THEXP_COMMIT_DISABLE # 用于禁止在 Experiment 运行时提交
1
2

如果你使用了Trainer，那么在 Params 中存在一个变量为 git_commit，将其值设置为False，会在 Trainer 初始化的时候添加禁止提交的环境变量。

# 运行环境判断

在作者做相关实验的过程中，存在一些需要判断本机或是远程环境，以及位于debug的需求，同时，基于这些需求，对一些超参数的设置也希望能够灵活判断，因此，thexp 框架内置了一些基本的运行环境变量以供判断，这包括：

  • 当前系统环境（win、linux、mac）
  • 本机环境还是远程环境
  • 是否由 Pycharm 运行，以及是否位于 Pycharm 的 Debug mode。

你可以通过以下的代码快速尝试您当前的环境。

from pprint import pprint
from thexp import ENV
pprint(ENV.__dict__)

# 参数规范

在 Meter 和 Params 中，提供了众多特性和语法糖，这是因为其内部通过Python中的魔法方法（Magic function），重定义了许多操作。而由于工程上的一些困难，要正常使用这些特性，其类内部的变量命名需要满足一条规范：

变量名前后不能有下划线（中间可以存在任意下划线）

如果存在下划线，在一部分情况下，程序会直接抛出异常；而在一些不方便抛出异常的情况下，这些类提供的特性可能无法正常应用在这些变量名对应的变量上。

# NoneItem

运行下面的代码，观察输出

from thexp import Meter
m = Meter()
print(m.k)
1
2
3

输出

NoneItem()
1

NontItem

任何原本不在meter中的变量在获取时不会报错，而是会返回一个 NoneItem 类型，该类型具有以下的特性：

  • noneitem == None
  • noneitem + any = any
  • noneitem - any = -any
  • noneitem * any = 1*any
  • noneitem / any = 1/any
  • 任何对象和 noneitem 比较大或者小均返回 True
  • int(noneitem) = 0, float(noneitem) = 0

该类位于 thexp.base_classes.trickitems，提供了一些奇技淫巧感兴趣可以看一下它的实现。该特性减少了某些情况下的代码量，比如说，可控的多段 loss 相加，如下面的代码：

all_loss = 0
if conditiona:
    all_loss = all_loss + a_loss
if conditionb:
    all_loss = all_loss + a_loss
1
2
3
4
5

如果使用meter，则会变成：

# all_loss = 0 # 可以不用写了
if conditiona:
    m.all_loss = m.all_loss + m.a_loss
if conditionb:
    m.all_loss = m.all_loss + m.a_loss
1
2
3
4
5

注意

注意 noneitem 做运算需要放在前面，否则执行方法时调用的不是noneitem内重写的运算符。

# 项目目录大文件后缀

在实验中临时保存在 repository 目录的文件，如果是文本文件，一般较小，Git 提交不存在问题；但如果存在一些较大的向量或模型文件，用 Git 提交它们会导致程序运行缓慢或程序目录过大。为了避免这些情况，thexp 对 .gitignore 文件做了补充，额外添加了以下后缀：

*.pth
*.npy
*.ckpt
1
2
3

当您在项目目录下存放以这些为后缀的文件时，这些文件将不会被 Git 添加和提交
