# 引言

在 Pytorch 中构建数据集生成 Dataloader 的时候，很多情况下的通用代码是这样的：

from torchvision.datasets import CIFAR10
from torch.utils.data import DataLoader

transform = ...
dataset = CIFAR10(...,transform,...)
dataloader = DataLoader(dataset,...)
1
2

当我们进行复现工作时，这样的构建简单明了，没有任何问题；但当我们构建我们自己的实验时，这样的形式可能并不是最优解。

对于一些使用者的方向而言，变换数据集的形式是常见的事情。在实验中，有时不仅仅需要提供 (xs, ys) 这样的数据形式，还可能需要构建 (xs,xs2,ys), (ids, xs, xs2, ... ,ys) 等多变的数据形式。

面临这种需求时，我们基本上都是通过构建数据集的子类来完成，对于每个 Dataset 子类，都需要重写一遍 __getitem__ 方法——繁琐而臃肿。

但实际上，数据集并不是最基本的形式，从数据集内还可以拆分出其他三个部分：

  • 输入
  • 标签
  • 序号

输入、标签、序号等可能来源于不同的数据集，但无论什么数据集，只可能存在这三种类别。

基于这种抽象，DatasetBuilder 实现了灵活到极致的数据形式的构建方式。

# 基本使用








 
 
 
 
 
 
 
 

from thexp import DatasetBuilder
from torchvision.datasets import CIFAR10

_dataset = CIFAR10(...)
xs, ys = _dataset.data, _dataset.targets

transform = ...
dataset = (
    DatasetBuilder()
        .add_inputs(xs).add_labels(ys) # 构建输入和标签的数据源
        .toggle_id()
        .add_x(transform=transform)
        .add_x(transform=transform)
        .add_y()
)

DatasetBuilder中的所有相关方法均支持链式调用。

默认值

当数据源只有一对输入和标签时候，可以直接通过 DatasetBuilder类的默认值输入


 






dataset = (
    DatasetBuilder(xs, ys)# 构建输入和标签的数据源
        .toggle_id()
        .add_x(transform=transform)
        .add_x(transform=transform)
        .add_y()

# 多数据源

你可以很容易的构建多数据源的数据集







 
 







xsa, ysa = _dataset.data, _dataset.targets
xsb, ysb = _dataset.data, _dataset.targets

transform = ...
dataset = (
    DatasetBuilder()
        .add_inputs(xsa).add_labels(ysa) # 默认数据源名字分别为 xs, ys
        .add_inputs(xsb, source_name='bxs').add_labels(ysb, source_name='bys') # 多数据源必须更改额外的数据源的名字
        .toggle_id()
        .add_x(transform=transform)
        .add_x(transform=transform, source='bxs') # 
        .add_y()
        .add_y(source='bys')
)

# 子集与假扩增

使用 DatasetBuilder 构建子集也同样方便，使用其 subset 方法即可：











 


from thexp.contrib.data import splits

train_idx, val_idx = splits.train_val_split(ys, val_size=10000)

dataset = (
    DatasetBuilder(xs, ys)
        .toggle_id()
        .add_x()
        .add_delegate(RandomDelegate(xs, ys), transform=...)
        .add_y()
        .subset(train_idx)
)

# 代理

单独使用 DatasetBuilder 能够满足大部分的需求，但总会有例外。以构建 Triplet 损失而言，正例 (positive) 和负例 (negative) 需要在数据集中另外选择，和锚样本 (anchor) 一起提供，这三者的下标互不相同，但DatasetBuilder 只有简单的按下标提供数据的功能——太过复杂的需求大多和具体任务有关，相对抽象的 DatasetBuilder 没有办法满足它们。

为了填补这一部分的缺口，thexp 额外提供了代理 (Delegate) 类。

DatasetBuilder 中存在代理时，会将需要获取的数据的下标传递给代理，代理根据具体使用者的实现返回相应的数据形式。

from thexp import Delegate
class RandomDelegate(Delegate):
    def __init__(self, xs, ys):
        super().__init__()
        self.xs, self.ys = xs, ys

    def __len__(self):
        return len(self.xs)

    def __call__(self, index, builder=None):
        pos_x = self.xs[index]
        neg_x = self.xs[np.random.randint(0,len(self.xs))]

        return self.X_(pos_x), self.X_(neg_x)


_dataset = CIFAR10(...)
xs, ys = _dataset.data, _dataset.targets
RandomDelegate(xs, ys)

dataset = (
    DatasetBuilder(xs, ys)
        .toggle_id()
        .add_x()
        .add_delegate(RandomDelegate(xs, ys), transform=...)
)

如何判断数据的排列顺序？

在 DatasetBuilder中，以 (下标，输入，标签) 的顺序提供数据。对于非代理数据，按照链式调用的顺序，将下标、输入、标签依次插入到相应的位置。代理数据在最后读取，同样按下标、输入、标签依次插入到相应的位置。

也即在上述的代码中，不管 add_x()/toggle_id()/add_delegate 的调用顺序如何更换，最终提供的数据形式均为 (ids, xs, pos, neg, ys, neg_ys)
