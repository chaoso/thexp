# 数据集整合：DataBundler

from thexp import DataBundler
1

该模块提供了两类（chain / zip）组织数据集的方法。

# Chain and Zip

DatasetBuilder 以 Dataloader 为单位，提供比 DatasetBuilder 更上层的数据集构建方式和方法。

DatasetBuilder 同样支持链式调用。

# chain

如果需要将两个 DataLoader 按顺序迭代其中的数据，一般情况下的最佳实践为：

from itertools import chain
for (xs,ys) in chain(dataloader_a,dataloader_b):
    ...
1
2
3

而通过 DataBundler ，代码如下：

from thexp import DataBundler
bundler = DataBundler().add(dataloader_a).add(dataloader_b)

for (xs,ys) in bundler:
    ...
1
2
3
4

# zip

如果需要将两个 DataLoader 同时读取，一般情况下的最佳实践为：

for (xsa, ysa), (xsb, ysb) in zip(dataloader_a,dataloader_b):
    ...
1
2

通过 DataBundler ，代码如下：

from thexp import DataBundler
bundler = DataBundler().add(dataloader_a).add(dataloader_b).zip_mode()

for (xsa, ysa), (xsb, ysb) in bundler:
    ...
1
2
3
4
5

# cycle zip

如果两个 DataLoader 的长度不同，如果想以最长的 DataLoader 为基准，循环加载较短的 DataLoader，那么在使用 zip 合并数据时的最佳实践一般为：

from itertools import cycle
for (xsa, ysa), (xsb, ysb) in zip(dataloader_a,cycle(dataloader_b)):
    ...
1
2
3

通过 DataBundler ，代码如下：

bundler = DataBundler().add(dataloader_a).cycle(dataloader_b).zip_mode()
for (xsa, ysa), (xsb, ysb) in bundler:
    ...
1
2
3

# Auto Device

如果只有这些功能，那么 DataBundler 只是通过链式调用稍微比原实现好用了一点， DataBundler 构建最初的想法，是节省每次对数据调用 data.to(device) 的代码，在加载数据的时候就自动分配设备。

只需要在构建 DataBundler 的时候调用 bundler.to(device) ，即可实现这一功能：

bundler = DataBundler().add(dataloader_a).cycle(dataloader_b).zip_mode().to(device)
