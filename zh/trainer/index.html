# 内部维护变量

在下面的示例中可以看到 Trainer 维护的这些变量，其中，saver/rnd/logger/writter 均使用懒加载，仅在使用时会请求 experiment 分配目录和生成文件。

from thexp import Trainer,Params
import random

class myTrainer(Trainer):
    pass

trainer = myTrainer(Params())

for i in range(50):
    trainer.logger.info(i)

for i in range(20):
    trainer.writter.add_scalar("test",random.random(),i)

# 使用Trainer构建训练流程

下面展示了一个简单的完整的训练流程

模型部份

import torch.nn as nn
class MyModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc = nn.Linear(28 * 28, 10)

    def forward(self, x):
        x = x.view(x.shape[0], -1)
        x = self.fc(x)
        return x

Trainer部份：定义一个类，并重写 callbacks/datasets/models/train_batch 方法

from thexp.frame import Meter, Params, Trainer
class MyTrainer(Trainer):
    def callbacks(self, params: Params):
        from thexp import callbacks
        callbacks.LoggerCallback().hook(self) # 添加日志输出功能

    def datasets(self, params: Params):
        from torchvision import transforms
        from torchvision.datasets import FakeData
        from thexp.torch.data.dataloader import DataLoader

        dataset = FakeData(size=32 * 10, image_size=(28, 28), transform=transforms.ToTensor())
        train_loader = eval_loader = test_loader = DataLoader(dataset, shuffle=True, batch_size=32, drop_last=True)

        self.regist_databundler(
            train=train_loader,
            test=test_loader,
            eval=eval_loader,
        )
        self.to(self.device)

    def models(self, params: Params):
        from torch.optim import SGD
        self.model = MyModel()
        self.optim = SGD(self.model.parameters(), lr=params.lr)
        self.cross = nn.CrossEntropyLoss()
        self.to(self.device)

    def train_batch(self, eidx, idx, global_step, batch_data, params, device):
        optim, cross = self.optim, self.cross
        meter = Meter()
        xs, ys = batch_data

        # 训练逻辑
        logits = self.model(xs)
        meter.loss = cross(logits, ys)

        # 反向传播
        meter.loss.backward()
        optim.step()
        optim.zero_grad()

        return meter


params = Params()
params.epoch=20
params.lr = 0.01

trainer = MyTrainer(params)
trainer.train()

# Auto device

在上面的代码示例中，可以看到重写的datasets()和models()方法最后都调用了 to(self.device)，该方法作用和 model.to(device)/tensor.to(device) 作用类似，且更加的简便，在训练过程中，无需再次声明或者为向量重新指定 device。

Trainer 中的 device 变量（torch.device） 默认生成于 Params 中的 device 变量（str）

该方法暂时还没有支持和测试多GPU下的情况

# 回调函数

thexp 为 Trainer 提供了一套回调机制，在之前的示例中，MyTrainer类重写了 callbacks() 方法，并在其中注册了 LoggerCallback，从而在训练过程中能够自动完成日志输出。如果不注册该回调类，则在大多数过程中程序都会保持静默。

提示

在 Trainer 的一些方法中，也有一些方法会主动的进行日志输出。

所有回调类的注册方法均通过 hook(trainer) 来完成，thexp提供了众多的回调类，包括自动保存、日志自动输出等，位于 thexp.frame.callbacks，您可以通过下面的语句直接导入并使用：

from thexp import callbacks
...
1
2

推荐默认 `callback`

按照 约定优于配置 的原则，callback 模块实现了许多和超参数中变量名相关的回调类，您可以根据需求酌情添加（如EMAUpdate或LRSchedule），或编写适合您自身试验的回调类。

但在这些模块中，仍然有几个比较通用的回调类，在默认情况下，添加会节省许多代码，这包括：

callbacks.LoggerCallback() # 用于对 train/test/eval 过程中的记录值进行日志输出
callbacks.AutoRecord() # 用于自动记录 train/test/eval 流程中返回的Meter中的值到 Tensorboard 中

# 更改实验名

上面的日志中，可以看到实验级别的目录名为 val.myExp 。其中，val 为Trainer 所在的文件名，myExp 则为 Trainer 的类名的前缀部份（去掉 Trainer 后缀，补上 Exp 后缀）。如果需要主动命名实验，则需要更改该类的类变量 __exp_name__：，此时实验名构造为 {模块文件名}.{__exp_name__}

class MyTrainer(Trainer):
    __exp_name__ = "DemoExp"
1
2

此时构造的实验名就会变成 val.DemoExp

提示

添加模块名是为了尽可能的保证实验名唯一。在大多数情况下，这种区分方式已经足以在默认（不改动）情况下区分开不同性质的实验名，但总有例外，请使用者注意区分。

# Params 支持

如之前提到的，为了更方便的使用，Trainer 对 Params 额外添加了一些功能支持，具体如下：

# device

Trainer 中的 device 变量（torch.device） 默认生成于 Params 中的 device 变量（str）

# 计数

Params 中默认在训练过程中更改以下三个 Params 变量

  • eidx # 以运行的 epoch 数，从1开始
  • idx # 当前 epoch 的 迭代次数，即经过了多少个 batch
  • global_step # 总迭代次数（总共经过了多少个 batch）

# initial

Trainer 初始化的最初会同时调用 Params 中的 initial() 方法，因此如果需要一些根据最终输入参数动态生成的参数，可以放在该方法中（如 Schedule）。

当然，单独使用并手动调用 initial() 方法也是可以的。
