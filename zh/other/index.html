# 其他

之前的几个篇幅介绍了 thexp 中的主体试验框架，希望您能赞同这是一个十分好用的框架并使用它。

本节将介绍这个实验框架之外的一些内容，这包括本框架其他的一些类与方法，提供的命令行支持，目录结构解释等，您可以以此通过导航定位阅读相关内容。

# ParamGrouper

这用于 torch 中的 Optimizer，针对不同类别参数构建不同 param_group(Optimzer 中更新参数的最大单位)。

from thexp.contrib import ParamGrouper
from thexp import Params

params = Params()
params.optim = params.create_optim('SGD', lr=0.01, momentum=0.9, weight_decay=5e-4)

grouper = ParamGrouper(modela, modelb, ...)
noptim = params.optim.args.copy()
noptim['weight_decay'] = 0
param_groups = [
    grouper.create_param_group(grouper.kernel_params(with_norm=False), **params.optim.args),
    grouper.create_param_group(grouper.bias_params(with_norm=False), **noptim),
    grouper.create_param_group(grouper.norm_params(), **noptim),
]

sgd = params.optim.build(param_groups)

# EMA

EMA（指数移动平均）是一种非常常见的提点方式，其原理就是额外维护一个和原模型一样的模型，并将每次训练后的原模型参数以某权重更新到该模型中。也即是说，模型训练方法还是原来的训练方法，但是在测试的时候用训练的每一步许许多多次模型的加权和来测试。

from thexp.contrib import EMA

model = ... # 用于 train
ema_model = EMA(model)

...

optim.step()
ema_model.step() # 仅用于 eval/test

# timing

一个临时用于计算各方法之间用时的模块

from thexp.utils import timeit

from time import sleep
timeit.start()
for i in range(10):
    timeit.mark("load")
    sleep(0.1)
    timeit.mark("load2")
    sleep(0.2)
    print(timeit)
timeit.end()
print(timeit.meter())

# 命令行支持

thexp 对一些常用命令提供了额外的命令行支持。

# reset/archive/delete

# 重置代码到某次试验时的代码
thexp reset <test_name>
thexp reset --test=<test_name>
thexp reset --test_name=<test_name>

# 打包某次试验时的代码
thexp archive <test_name>
thexp archive --test=<test_name>
thexp archive --test_name=<test_name>

# 直接删除某实验（一般为结束后删除某失败无价值的试验）
thexp delete <test_name>
thexp delete --test=<test_name>
thexp delete --test_name=<test_name>

# Tensorboard

Tensorboard有时启动会存在临时目录的问题，此外每次都需要定位目录，thexp 额外提供了基于 testname 的定位方式，除此之外，其余参数均和原 tensorboard 命令的参数相同（由于参数解析的问题，该方式不能输入单横杠参数）：

thexp board [--logdir=<logdir>]
thexp board [--test=<test_name>] # find test_name and tensorboard it 
thexp board  # default open ./board

# 深度学习试验模板生成

效果约等于 git init ，但是会在相同目录下生成额外的诸多 py 文件。

thexp init
1

该模板通过合理的组织形式，能够在不断的补充和重改试验中保持住项目整体的代码复杂度，清爽简洁，适用于较长实验周期的项目代码。

对模板内具体内容的介绍请参考 Cookbook-Structure

# 目录组织

thexp 的各模块按照如下的方式进行区分：

  • analyse : 用于对基于 Experiment 完成的试验进行分析和可视化
  • base_classes : 提供了很多基类和数据结构
  • contrib : 对pytorch中许多功能的一些扩展和补充
  • calculate : 用于对向量、存储空间和数值等计算的方法或类
  • decorators : 各种用途的装饰器
  • frame : 所有介绍的框架相关功能均位于该模块内
  • future : 以后打算添加但由于各种原因还没有真正添加进去的功能，bug
  • nest : 对 thexp 中框架功能的一些扩展和补充（mixin）
  • util : 一些基本的功能库
