# 目录结构

可以看到基础的目录只有三个： 用于处理和提供数据有关的data，用于提供模型结构的arch，以及用于训练的trainer

# arch

arch(architecture) 目录下建议放置最基础的模型文件，一个文件一个模型即可，没有其他更多的需求或变动，这也是最常用的方式。

# data

data 目录下会有许多内容，这包括

  • 数据源的提供（dataxy.py）
  • 数据增广方法（transform.py)
  • thexp 中的 delegate 构建（delegate.py）

# 数据源(dataxy)

数据源中已经写好了一个 cifar10 数据集的示例，这里建议的写法是按照元组的形式提供样本和标签，而不是以 torch 中 Dataset 的形式，这样配合 thexp 提供的 DatasetBuilder，就能够以更灵活的形式构建数据集。

# 数据增广方法

大部分增广方法都是通过多个增广方法组合而成，如计算机视觉领域常用的随机翻转裁切的增广：

from torchvision import transforms
class Weak(object):
    def __init__(self, mean=None, std=None):
        lis = [
            transforms.RandomHorizontalFlip(),
            transforms.RandomCrop(size=32,
                                  padding=4,
                                  padding_mode='reflect'),
            transforms.ToTensor(),

        ]
        if mean is not None and std is not None:
            lis.append(transforms.Normalize(mean=mean, std=std))
        self.weak = transforms.Compose(lis)

    def __call__(self, x):
        return self.weak(x)

# 编写即运行

参考 trainer 文件夹初始化后生成的 _trainer.py 文件的代码，您可以发现在开头和结尾进行了程序入口（main）判断，并进行了相应的切换目录等操作。这意味这这样的文件可以写好了直接运行，省去了在项目下另见一个文件来 run 的操作。

# 多继承/Mixin

通过 Python 的多继承机制，相似试验之间的相同功能可以本很好的拆分出来，以类的方式构造，并通过多继承的方式（Mixin机制）融入到各个 Trainer 中

您可以看到 trainers/mixin 目录下，已经分门别类的放好了各种用途的 Mixin ，并在 _trainer.py 中进行了多继承：

class TempleteTrainer(callbacks.BaseCBMixin,
                      datasets.TempleteDatasetMixin,
                      models.TempleteModelMixin,
                      acc.ClassifyAccMixin,
                      losses.CELoss,
                      tricks.TrickMixin,
                      Trainer):
